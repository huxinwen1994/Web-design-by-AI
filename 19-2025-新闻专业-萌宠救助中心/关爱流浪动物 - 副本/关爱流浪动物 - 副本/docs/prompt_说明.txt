# Prompt 使用说明文档

## 一、初始任务 Prompt

### 1.1 项目启动Prompt
"我需要创建一个关爱流浪动物的信息交互类网站，名称为'萌宠救助中心'。网站的主要目标是：
1. 展示待领养的流浪动物信息
2. 介绍领养流程和机构信息
3. 招募志愿者
4. 接受捐助并公示使用情况

网站需要包含以下页面：
- 首页：展示机构简介、推荐领养动物、救助故事、公告
- 关于我们：机构详细介绍
- 待领养：动物列表，支持按类型、年龄、体型筛选
- 志愿者：招募说明和报名表单
- 捐助支持：捐助方式和公示

技术要求：
- 使用HTML5语义化标签
- CSS外部样式表，使用Flexbox或Grid布局
- 响应式设计，支持桌面和移动端
- 使用原生JavaScript实现交互功能（轮播图、筛选、表单验证）
- 设计风格：温馨、清新、以绿色为主色调"

### 1.2 页面结构Prompt
"请为首页创建以下结构：
1. 导航栏：包含logo、网站名称、主导航菜单、搜索框、登录/注册按钮
2. 横幅轮播区：左侧轮播图（3张）带左右切换和圆点指示器，右侧文字介绍
3. 三栏内容区：本周推荐领养、救助故事、中心公告
4. 快速链接区：两个卡片引导用户了解领养流程和捐助方式
5. 页脚：机构信息和联系方式

请使用清晰的HTML语义化标签，并添加必要的class名称用于样式。"

### 1.3 样式设计Prompt
"请为网站创建CSS样式文件，要求：
1. 创建common.css作为公共样式，包含：
   - CSS Reset
   - 导航栏样式（固定宽度1180px居中，绿色主题）
   - 卡片样式（白色背景，圆角，阴影）
   - 按钮样式（绿色主题，悬停效果）
   - 页脚样式（渐变绿色背景）

2. 创建index.css用于首页特定样式：
   - 轮播图容器（左右布局）
   - 三栏卡片布局（使用float或flexbox）
   - 动物列表样式（图文混排）

3. 使用Flexbox实现布局
4. 添加基本响应式：在768px以下调整为单列布局
5. 配色方案：主色#4CAF50（绿色），背景#E8F5E9（浅绿），文字#333"

## 二、迭代过程 Prompt

### 2.1 轮播图功能优化
"请为首页横幅创建轮播图JavaScript功能：
1. 自动播放，每4秒切换一次
2. 点击左右箭头手动切换
3. 点击圆点可跳转到对应图片
4. 鼠标悬停时暂停自动播放，移开后继续
5. 使用原生JavaScript实现，不依赖jQuery
6. 确保代码封装在IIFE中，避免全局污染"

### 2.2 待领养页面筛选功能
"为adoption.html页面添加动物筛选功能：
1. 在页面顶部有筛选栏，包含：类型（全部/猫咪/狗狗）、年龄（幼年/成年/老年）、体型（小型/中型/大型）
2. 为每个动物卡片添加data属性标记其属性（data-type, data-age, data-size）
3. 创建filter.js实现筛选逻辑：
   - 点击筛选项时添加active类
   - 根据选中的条件显示/隐藏对应的动物卡片
   - 支持多条件组合筛选
4. 使用原生JavaScript，不使用框架"

### 2.3 搜索功能实现
"为导航栏的搜索框添加功能：
1. 输入关键词后按回车或点击搜索按钮触发搜索
2. 在当前页面高亮显示包含关键词的内容
3. 如果是在adoption页面，筛选出名称或描述包含关键词的动物
4. 显示搜索结果数量
5. 添加防抖处理，避免频繁触发
6. 使用原生JavaScript实现"

### 2.4 表单验证功能
"为volunteer.html和donate.html页面的表单添加验证功能：
1. 必填项检查（姓名、联系方式、邮箱等）
2. 邮箱格式验证
3. 手机号格式验证（11位数字）
4. 实时验证提示（失去焦点时验证）
5. 提交时统一验证，验证失败显示错误提示
6. 验证成功显示提交成功提示
7. 使用原生JavaScript，不依赖插件"

### 2.5 响应式布局优化
"优化网站的响应式设计：
1. 在common.css中添加媒体查询
2. 断点设置：
   - 桌面端：> 1024px（正常显示）
   - 平板端：768px - 1024px（调整间距和字体）
   - 移动端：< 768px（单列布局）
3. 移动端优化：
   - 导航菜单改为汉堡菜单
   - 三栏布局改为单列
   - 搜索框宽度自适应
   - 轮播图左右布局改为上下布局
4. 确保所有交互功能在移动端仍然可用"

### 2.6 代码规范化
"请优化代码结构：
1. 将分散的JavaScript代码整合到main.js文件中
2. 使用模块化结构，每个功能封装为独立函数
3. 添加必要的注释说明
4. 统一变量命名规范（驼峰命名）
5. 移除重复代码
6. 确保所有外部资源（CSS、JS）正确引用"

### 2.7 内容展开/折叠功能
"在donate.html页面的捐赠公示部分添加展开/折叠功能：
1. 默认只显示最近3条记录
2. 点击'查看更多'按钮展开所有记录
3. 展开后按钮文字变为'收起'
4. 添加平滑的展开/收起动画
5. 使用原生JavaScript实现"

### 2.8 动物详情页优化
"优化adoption-detail.html页面：
1. 添加动物照片轮播图
2. 使用表格展示动物详细信息（年龄、性别、体型、健康状况等）
3. 添加'我要领养'表单弹窗
4. 添加分享按钮
5. 使用卡片布局组织内容"

## 三、个人总结

### Prompt驱动开发的实践体会

通过本次使用大型语言模型辅助开发的实践，我深刻体会到了AI在编程中的强大辅助作用，同时也认识到人工智能工具的局限性。

**Prompt设计的重要性**：
初期我发现，越是详细、结构化的Prompt，生成的代码质量就越高。模糊的需求描述往往导致代码需要大量修改。因此我学会了将需求分解为具体的功能点，明确技术要求和实现细节。

**迭代优化的必要性**：
首次生成的代码往往不能完全满足需求，需要通过多轮Prompt进行迭代优化。例如轮播图功能，我先请求基础的切换功能，然后逐步添加自动播放、暂停、圆点跳转等细节，这种渐进式的开发方式更容易把控质量。

**遇到的问题**：
1. AI生成的代码有时存在逻辑错误或兼容性问题，需要人工调试
2. 样式细节需要反复调整才能达到理想效果
3. 多个功能之间的整合需要手动处理
4. 响应式设计的断点和适配需要多次测试

**Prompt模式的优点**：
1. 大幅提升开发效率，快速搭建项目框架
2. 学习新技术和最佳实践的好帮手
3. 代码规范性较好，注释清晰
4. 能快速实现复杂功能的原型

**反思与展望**：
AI辅助编程是一种强大的工具，但不能完全依赖。开发者需要具备基本的编程知识和调试能力，才能充分发挥AI的价值。未来我会更加注重Prompt的质量，同时加强自身的代码能力，实现人机协作的最佳效果。
