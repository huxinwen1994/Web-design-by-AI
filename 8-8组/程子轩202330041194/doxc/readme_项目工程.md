1. 项目名称与简介
项目名称：大学生早餐习惯可视化网站项目简介：本项目是一款基于高校大学生早餐消费习惯调研数据的单页可视化网站，通过图表、交互组件等形式，直观呈现大学生早餐的消费类型、金额分布、食用场景等信息，并输出针对性建议。网站采用 “校园清新 + 早餐主题” 风格，兼顾数据展示的清晰性与交互体验的趣味性。
2. 目标用户与应用场景
目标用户：
高校后勤管理者：用于优化校园早餐供给方案；
大学生群体：用于了解自身早餐习惯的合理性；
校园调研人员：用于展示调研数据成果。
应用场景：
校园后勤部门分析早餐消费趋势的参考工具；
大学生健康饮食科普的轻量化展示载体；
调研项目的数据可视化成果交付物。
3. 网站信息结构与主要板块说明
网站为单页架构，含 “顶部导航栏 + 3 个核心版块”：
顶部导航栏：含 “调研背景”“数据可视化”“结论建议” 3 个导航项，支持平滑跳转至对应版块；
调研背景版块：展示调研对象、样本量、维度等基础信息，以列表形式呈现；
数据可视化版块：通过饼图、柱状图、折线图展示早餐类型占比、年级消费金额分布、外带率等数据，配套表格适配移动端；
结论与建议版块：含核心结论列表、分对象建议（学生 / 食堂）、早餐建议反馈表单。
4. 主要交互功能介绍
导航栏交互：点击导航项平滑滚动至对应版块，页面滚动时自动切换导航项激活态；
图表交互：鼠标悬停图表时显示数据详情（如早餐类型 + 占比 + 特点）；
核心结论交互：结论条目带黑色细线框，hover 时轻微放大、离开时渐缩；
建议版块交互：“给学生”“给校园食堂” 标题支持点击展开 / 收起对应内容；
表单交互：反馈表单提交时验证必填项与字数，通过后弹出成功提示并重置表单。
5. Prompt 使用情况简述
本项目采用 “分模块 Prompt 驱动开发” 模式：
按 “页面结构→导航样式→数据可视化→核心结论交互→建议版块交互” 的模块拆分需求；
每个模块撰写精准 Prompt，明确技术栈（原生 JS/CSS）、视觉风格（早餐主题）、交互逻辑（如 hover 缩放、点击展开）；
整合 AI 输出的代码后，微调样式兼容性与交互细节，实现需求落地。
6. 网站运行与测试说明
运行说明：无需依赖服务器，直接打开项目根目录下的index.html文件即可在浏览器中运行；
测试说明：
浏览器兼容性测试：在 Chrome、Edge、Firefox 主流浏览器中验证页面渲染与交互功能；
设备适配测试：在桌面端（1200px+）、平板（768px）、手机（480px）尺寸下，验证布局与组件适配性；
交互功能测试：依次测试导航跳转、图表 hover、结论缩放、建议展开、表单提交等功能的有效性。
