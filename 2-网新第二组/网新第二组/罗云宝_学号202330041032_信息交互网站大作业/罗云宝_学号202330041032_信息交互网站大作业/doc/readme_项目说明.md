湖师大 AED 急救资源查询平台・校园生命守护站 项目说明文档
一、项目名称与简介
1. 项目名称
湖师大 AED 急救资源查询平台・校园生命守护站
2. 项目简介
本项目是面向湖南师范大学全体师生及教职工的校园 AED 急救资源综合服务平台，由校后勤保障处与校医院联合建设。平台整合了校内 86 台 AED 设备的精准点位、急救数据统计、使用指南、培训报名等核心功能，通过可视化的数据图表、交互式校园地图及实时热点资讯，构建起 “查询 - 学习 - 应急 - 培训” 一体化的校园急救服务体系，实现了教学区、生活区、运动场馆 AED 设备 100% 覆盖的信息同步，为校园生命安全提供数字化保障。
二、目标用户与应用场景
1. 目标用户
核心用户：湖南师范大学全体本科生、研究生、教职工
辅助用户：校医院医护人员、后勤保障工作人员、校园急救志愿者
2. 应用场景
日常查询：师生课前、运动前查询周边 AED 设备的具体位置与获取方式
应急处置：突发紧急医疗状况时，快速定位就近 AED 设备、调取操作指南并联系校医院急诊
知识学习：通过平台内置的操作教程、急救案例，学习 AED 使用及校园急救相关知识
培训参与：查看校园 AED 培训动态，在线报名参与急救实操培训
管理运维：后勤与校医院工作人员查看 AED 设备分布、维护记录及校区覆盖数据
三、网站信息结构与主要板块说明
网站采用分层级的信息架构，整体分为导航栏、封面、热点推荐、正文板块、页脚五大模块，具体如下：
1. 导航栏
分为上下两栏，承担全局导航与快捷功能入口作用：
第一栏：左侧为平台标题（搭配 AED 简笔图标与校徽简化元素）；右侧设校内场所搜索框、校医院急诊（88879120）/120 急救电话快捷拨打图标、微信分享图标、实时日期及 “校园培训更新提醒” 小红点
第二栏：包含 “首页、校园 AED 地图、数据动态、使用指南、校园急救案例、培训报名、资源下载、联系我们” 八大核心功能板块
2. 封面
以全屏轮播图为核心展示载体：
轮播内容：涵盖校园 AED 设备实拍、投放标识、培训实操、医护演示、校区覆盖率热力图 5 类核心场景
交互与标识：轮播切换带 “心跳波形” 过渡动画与校徽淡入淡出效果，指示点替换为微型 AED 图标；底部叠加半透明数据条，展示 AED 设备收录量及覆盖范围（2024.10 更新）
3. 热点推荐（固定底边栏）
固定于页面底部的急救资讯轮播栏，背景为 80% 透明度的急救红并叠加心跳线与校园建筑暗纹：
内容类型：包含 AED 新增投放、急救演练、政策通知、案例分享、培训预告 5 类资讯，每条资讯搭配专属图标
交互特性：6 秒自动轮播，支持手动左右切换，鼠标悬停可暂停轮播
4. 正文板块
共设四大核心正文板块，覆盖数据、指南、地图、培训全场景：
板块一：AED 核心数据概览：2:3 分栏布局，左侧为医疗蓝竖排标题栏，右侧以 2×2 网格展示校区分布柱状图、场所覆盖率环形图、年度增长趋势折线图、急救响应效率数字卡片，支持图表元素点击查看细分数据
板块二：校园 AED 场景化信息栏：三栏均等布局，分别为 AED 使用指南（含四步操作示意图与专属注意事项）、校园急救案例（3 个校医院认证真实案例）、校园政策与投放计划（最新政策与校区投放进度图）
板块三：交互式校园 AED 地图：核心功能板块，顶部设校区切换与场所类型筛选入口；左侧 70% 为基于 Leaflet 的交互式地图，支持 AED 点位悬停查基础信息、点击看详情（含获取方式、周边应急资源、导航路线）；右侧 30% 为校区 AED 数据卡片，支持点击定位对应校区
板块四：师生认知与培训报名：3:2 分栏布局，左侧展示师生 AED 认知度条形图、使用障碍调研饼图；右侧为培训动态列表与报名表单，表单支持实时验证与报名成功反馈
5. 页脚
整合平台服务与备案信息：
核心入口：网站地图、关于我们、培训报名、数据反馈、隐私政策
联系方式：校医院急诊、后勤咨询电话、反馈邮箱及数据合作单位
底部标识：数据更新说明、AED 设备与校徽暗纹背景、贯穿页面的红色短线（中间嵌微型 AED 图标）
四、主要交互功能介绍
响应式适配功能：支持 PC（≥1200px）、平板（768-1199px）、手机（≤767px）多终端，移动端自动将多栏布局转为上下布局，地图适配手势缩放，导航栏折叠为汉堡菜单
交互式地图功能：支持场所搜索、校区 / 场所类型筛选、点位悬停 / 点击查信息、校园路线文字指引，移动端可触发导航示意图弹窗
数据图表交互：所有图表支持按校区、场所类型、时间筛选，点击图表元素可弹出具体数值与占比提示框，移动端支持触摸交互
表单与培训交互：培训报名表单支持实时必填项验证，提交后生成报名序号并短信推送须知；数据反馈表单可通过页脚入口弹窗填写
细节交互：导航栏菜单项 hover 显示红色下划线、卡片 / 按钮 hover 加阴影 / 轻微放大、点击触发彩色气球特效、返回顶部按钮支持平滑回滚
五、Prompt 使用情况简述
本项目前端代码的生成与优化全程基于 Prompt Engineering 实现，分为初始 Prompt与迭代 Prompt两个阶段：
初始 Prompt：以结构化指令明确网站框架、设计要求（色彩、字体、视觉元素、交互逻辑）、技术栈（Chart.js、Leaflet.js），指令包含详细的模块布局、样式参数、数据可视化需求，确保 AI 生成的代码符合校园急救平台的功能与视觉定位
迭代 Prompt：针对初始代码的技术难点进行定向优化，共完成 3 轮迭代
迭代 1：替换地图瓦片源为高德地图，构建校区数据对象并编写renderMapData函数实现多校区切换
迭代 2：完善initCharts函数初始化图表，配置指定配色的横向柱状图与甜甜圈图，添加全局点击气球飘浮特效
迭代 3：实现 AED 重要性说明模态框逻辑，完善培训报名表单的onsubmit事件，添加表单验证与提交成功提示
六、网站运行与测试说明
1. 技术栈依赖
前端基础：HTML5、CSS3、原生 JavaScript
数据可视化：Chart.js（图表渲染）
地图组件：Leaflet.js + 高德地图瓦片源 + 湖师大校园 GeoJSON 数据
样式规范：CSS 变量（急救红 #D32F2F、医疗蓝 #1976D2 等核心色值）
2. 运行环境要求
浏览器：Chrome、Firefox、Edge 等现代浏览器（建议版本≥90），移动端支持 Safari、微信内置浏览器
网络环境：需支持高德地图瓦片源、Chart.js/Leaflet.js CDN 资源加载，校内网络环境优先
3. 核心测试要点
响应式测试：验证 PC / 平板 / 手机端布局适配性，重点测试导航栏折叠、地图手势缩放、表单适配效果
功能交互测试：检查 AED 地图点位查询、校区切换、图表数据筛选、培训表单提交的功能完整性与准确性
数据准确性测试：核对 AED 设备数量、校区分布、急救响应效率等核心数据与校后勤 / 校医院提供的原始数据一致性
应急功能测试：验证校医院 / 120 快捷拨打按钮的触发逻辑，确认 AED 使用指南视频弹窗可正常播放
