1. 项目名称与简介 
    项目名称：欢迎来到2024巴黎奥运会 ᕦ(ò_óˇ)ᕤ
    简介：该项目是一个专注于奥运会相关内容的信息平台，旨在为用户提供最新的奥运资讯、赛事动态和运动员风采。用户可通过平台查看奥运概况、精彩瞬间（闪耀时刻）、资讯集锦及直播/回放内容，感受奥林匹克魅力并为中国奥运健儿加油。
2.目标用户与应用场景 
    目标用户：对巴黎奥运会感兴趣的各类人群，包括体育爱好者、关注奥运赛事的观众、希望了解运动员动态的用户等
    应用场景：用户需要查询巴黎奥运会相关信息时、回顾比赛精彩瞬间时、获取最新赛事资讯时、观看赛事直播或回放时以及参与互动时，可通过该网站实现需求，支持在电脑、移动设备等多种终端访问 
3.网站信息结构与主要版块说明 
    网站整体结构包括导航栏、主要版块、页脚及右侧浮动按钮
    导航栏：
        包含平台logo、核心导航链接（首页、奥运概况、闪耀时刻、资讯集锦、直播/回放）及搜索框，用于快速跳转至各功能页面和信息检索
    主要内容区：
        ·首页：核心为导航入口（含 logo、搜索框及五大板块链接）、欢迎展示区（主题文字、代表团合照、功能跳转按钮），搭配页脚反馈入口与右侧浮动互动按钮（主题切换、喝彩、留言），作为网站总入口引导用户探索
        ·奥运概况：聚焦基础信息呈现，包含2024巴黎奥运会核心信息、赛事规模数据、中国代表团介绍，直观传递奥运及中国参赛相关概况
        ·闪耀时刻：以夺金瞬间为核心，设有各国实时奖牌排行榜，配有图文卡片，展现热门夺冠瞬间与运动员高光时刻
        ·资讯集锦：以资讯汇总为核心，设赛程预告、专题报道板以及群像图片，满足用户获取奥运相关新闻的需求
        ·赛事直播：围绕直播功能，分为“今日直播”“正在直播”“我的预约”，提供视频播放区、实时数据展示，搭配基础播放控制与预约直播等功能
    页脚：
        位于页面底部，包含 “意见反馈” 入口和版权信息（© 2024 奥运观察 版权所有），提供用户反馈渠道和版权声明
    右侧浮动按钮：
        包含四个功能按钮，分别为“返回顶部”（图标：向上箭头） “切换主题”（图标：调色板）、“为奥运健儿喝彩”（图标：奖杯）、“留言板”（图标：评论），提供便捷的互动入口。
4.主要交互功能介绍
    导航跳转：点击导航栏中的链接（如“奥运概况”“资讯集锦”），可跳转至对应页面
    搜索功能：通过搜索框输入内容，结合搜索图标，可检索奥运相关资讯，功能需依赖后端或JavaScript实现
    页面跳转：首页 “查看更多” 按钮点击后，跳转至 “奥运概况” 页面。
    互动按钮：右侧浮动按钮分别支持切换网站主题、触发为健儿喝彩的交互、打开留言板功能
    意见反馈：点击页脚 “意见反馈” 可触发反馈入口，进入弹窗可输入反馈意见并提交
5.Prompt 使用情况简述 
    一、核心 Prompt（指令）解析 
        本次任务的核心 Prompt 是一套完整的“奥运观察新闻专题网站开发需求指令”，由网页结构设计指令、交互功能实现指令、技术规范约束指令三大类子指令构成的组合式需求集合，明确了网站开发的目标、具体要求及执行标准，是后续所有开发动作的核心依据
    二、Prompt 使用依据 
        1. 网页结构类指令：以 Prompt 中“5个主要页面的布局规则、元素配置、内容呈现要求”为依据，以“导航栏需包含5个居中板块+左侧奥运五环logo”“首页需依次呈现欢迎文字、介绍文本、代表团照片、两个功能按钮”等为具体指令，作为页面结构搭建的直接依据
        2. 交互功能类指令：以 Prompt 中明确的交互规则为依据，例如“喝彩按钮点击喷射彩带”“添加图片轮播、悬停效果、菜单导航等交互元素”“整体红白配色，风格青春活力”等指令，作为交互逻辑开发的核心依据
        3. 技术规范类指令：以 Prompt 中“排版、配图、样式、响应式、交互实现方式”的约束为依据，例如“所有样式写入外部CSS”“配图需带alt属性”等指令，作为技术落地的强制依据
6. 网站运行与测试说明 
    运行环境：
        需在支持HTML5、CSS3和JavaScript的现代浏览器（如Chrome、Firefox、Edge等）中运行，依赖外部资源包括：本地CSS文件（`css/style.css`）、本地JavaScript文件（`js/main.js`）及本地asset文件（提供图片资源）
    测试要点： 
        (1)导航链接跳转是否正常，确保各页面路径正确
        (2)交互按钮功能是否生效（如 “查看更多” 跳转、浮动按钮点击反馈）
        (3)响应式布局测试：通过调整浏览器窗口大小，验证在不同设备（手机、平板、桌面）上的显示效果是否符合预期
        (4)外部资源加载：检查 Font Awesome 图标是否正常显示，无缺失或错误
        (5)搜索功能：测试输入内容后是否能正确触发搜索逻辑
