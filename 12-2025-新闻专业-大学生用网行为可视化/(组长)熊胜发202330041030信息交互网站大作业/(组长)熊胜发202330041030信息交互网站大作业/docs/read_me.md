大学生用网行为可视化・数字校园洞察站 - 项目说明文档
一、项目名称与简介
项目名称
大学生用网行为可视化・数字校园洞察站
项目简介
本项目是一款聚焦大学生用网行为的数字化平台，核心目标为三大维度：一是直观展示校园网络流量分布、应用偏好、时段特征等核心数据；二是创新提供校园网络 “潮汐” 现象模拟工具，支持场景回溯、预测及自定义压力测试；三是实现用网习惯自评、典型案例参考与互动调研功能，助力用户认知自身用网状态，为高校网络优化与学生健康用网提供数据支撑。
二、目标用户与应用场景
目标用户
在校大学生：需自查用网习惯、获取网络避堵建议、反馈网络使用体验的核心用户；
高校网络管理部门：需分析网络流量规律、优化网络资源分配、应对拥堵问题的管理用户；
教育及社会研究人员：需研究大学生用网行为特征、开展相关课题调研的专业用户。
应用场景
学生端：通过自评表单检测用网沉迷程度、参考 “避堵攻略” 选择优质网络时段 / 区域、查看典型案例调整自身用网习惯；
管理端：借助核心数据与潮汐模拟结果，针对性优化宿舍、教学楼、食堂等区域的网络配置，合理分配带宽资源；
研究端：基于平台聚合的用网行为数据，分析大学生用网模式与学习、生活的关联关系。
三、网站信息结构与主要板块说明
1. 导航区域
顶部栏：左侧为 “广播塔图标 + 网站名称” 的品牌标识，右侧为实时网速显示、迷你趋势图表占位、流量预警按钮（带红色闪烁提示）的状态面板；
主导航栏：包含 “首页、核心数据、潮汐实验室、习惯自评、互动调研”5 个锚点链接，支持激活态与 hover 态样式切换。
2. 封面轮播区
含 2 个自动轮播幻灯片（间隔 8 秒），底部设圆点指示器：
幻灯片 1：校园区域网络流量全天候热力图，区分宿舍（夜间高峰）、食堂（饭点高峰）、教学楼（日间高峰）的流量密度；
幻灯片 2：学生群体日均上网时长雷达图，覆盖社交、娱乐、学习、购物、其他五大维度。
3. 核心数据概览区
采用 “左侧垂直标题栏 + 右侧 2 列网格浮动卡片” 布局：
垂直标题栏：竖排显示 “核心统计” 文字；
4 个数据卡片：各时段流量分布（面积图）、应用类型偏好（环形图）、月度流量趋势（折线图）、网络健康指数（文本展示核心指标），卡片支持 hover 浮动效果。
4. 校园网络 “潮汐” 模拟实验室区
左右分栏布局（左侧控制台 + 右侧可视化区），核心为网络潮汐现象模拟：
控制台：支持场景回溯与预测、避堵攻略展示、自定义压力测试功能；
可视化区：含校园流场地图（涟漪效果展示拥堵度）、压力测试图表（运行后显示网速波动折线图）。
5. 场景分析与自评区
3 列网格布局：
上网习惯自评表单：5 道核心题目，提交后生成等级化自评报告；
典型案例库：5 个案例卡片，点击可查看详情；
互动调研：展示用网满意度雷达图，支持反馈输入与提交。
6. 底部区域
固定滚动提示条：展示校园热点信息；
页脚：包含服务电话、数据更新时间。
四、主要交互功能介绍
实时数据模拟：实时网速每 2 秒随机更新（10-20MB/s），更新带颜色闪烁提示；流量预警功能支持剩余流量实时递减（初始 10GB，速率 2GB/min）。
模态框交互：统一支持 “关闭按钮 + 遮罩层点击关闭”，覆盖流量预警、自评报告、案例详情三大场景。
潮汐实验室交互：预设 3 个典型场景，切换后校园流场地图拥堵度实时变化；自定义压力测试中，接入人数与网速基准值、波动幅度呈联动关系（人数越多，网速越低、波动越大）。
自评与案例交互：自评表单提交后，系统按分值判定 “深度沉迷 / 沉迷 / 良好 / 健康”4 个等级并生成个性化建议；案例卡片点击弹出详情模态框，调研区支持反馈输入与提交。
通用交互：卡片 hover 时上浮 + 阴影加深，流量预警按钮带红色闪烁动画，潮汐模拟采用涟漪效果，模态框实现淡入淡出过渡。
五、Prompt 使用情况简述
本项目采用 “基础蓝图 + 迭代优化” 的 Prompt 驱动开发模式，具体使用情况如下：
初始 Prompt：提供项目完整蓝图，包含网站主题、核心目标、页面结构、样式要求、技术栈等基础信息，确保开发方向准确；
迭代 1 Prompt：聚焦功能完整性增强，新增实时流量消耗模拟器、迷你趋势图表、卡片 hover 效果等功能，完善自评系统评分逻辑与案例查看功能；
迭代 2 Prompt：核心为创新模块开发，设计并实现 “校园网络‘潮汐’模拟实验室”，替换原有普通数据展示区，明确模块功能、布局与技术要求；
迭代 3 Prompt：侧重代码结构优化与工程化，要求分离 HTML、CSS、JavaScript 文件，模块化重构潮汐实验室，优化响应式布局与可访问性；
迭代 4 Prompt：聚焦细节完善与专业度提升，优化可视化配色、添加图表面积渐变效果、增强交互反馈，完善数据模拟算法。
六、网站运行与测试说明
运行环境要求
技术栈依赖：基础技术为 HTML5+CSS3+JavaScript（原生），可视化功能依赖指定图标（广播塔、时钟等）；
浏览器兼容性：支持 Chrome、Edge 等现代浏览器，暂不兼容低版本 IE 浏览器；
设备适配：支持 PC 端及移动端，需保证屏幕分辨率≥320px。
测试要点
布局测试：验证响应式适配效果（屏幕 < 992px 时，网格布局切换为 1 列），模块内边距、最大宽度居中显示是否符合规范；
功能测试：
实时数据模拟：网速更新、流量递减是否正常；
潮汐实验室：场景切换、压力测试的联动逻辑是否准确；
自评系统：评分计算、等级判定、建议生成是否正确；
模态框：所有场景下的打开 / 关闭功能是否正常；
视觉与交互测试：动画效果（卡片 hover、涟漪、闪烁）是否流畅，颜色搭配是否符合主题规范，交互反馈是否及时。
运行方式
解压项目压缩包，确保目录结构完整（index.html、css/style.css、js/main.js 等文件位置正确）；
直接用支持的现代浏览器打开 index.html 文件，即可启动网站；
启动后可通过主导航栏切换各功能模块，按页面提示操作交互功能。
