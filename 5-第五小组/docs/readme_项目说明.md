# 项目说明文档

## 一、项目名称与简介

**项目名称：Flavor Quest —— 中国菜系探索平台**

一个面向年轻人（特别是大学生、上班族）的美食探索网站，集中国地方菜系介绍、美食推荐、食谱学习、食材搭配、趣味互动于一体，强调“好看、好玩、好吃、有文化”。

旨在通过结构化的信息展示与简单直观的交互设计，帮助用户了解不同地区的代表性菜系及其经典菜品，并学会做一些菜品。网站以图文结合的方式呈现内容，强调信息的清晰组织与良好的浏览体验，同时通过 JavaScript 实现基础交互功能，使用户能够主动探索和切换信息，而非被动阅读。

## 二、目标用户与应用场景

**目标用户**主要为对美食文化感兴趣且想自己做菜的普通用户，以及希望快速了解不同地区菜系特点的初级探索者。

**应用场景**包括：

餐饮专业学生可在线学习八大菜系理论知识
厨房新手可查找简单食谱并学习基础烹饪技巧
聚会前可快速浏览地方特色菜寻找宴会菜品
美食评论家快速查阅菜系背景和历史渊源
美食爱好者下班后浏览放松、发现新菜品
采购食材前查看菜品所需原料和搭配建议
海外文化机构展示中国饮食文化多样性
跨文化交流中作为中国文化的直观展示

## 三、网站信息结构与主要板块说明

网站整体采用多页面结构，通过顶部导航栏实现页面之间的跳转，信息层级清晰，逻辑明确。

主要结构包括：

1. **首页（index.html）**
   作为网站入口页面，对项目主题进行整体展示，引导用户进入具体菜系内容页面。

2. **地方菜系页面（regional.html）**
   不同页面分别介绍不同地区或类型的菜系，通过卡片式布局呈现菜系名称、图片及简要说明，方便用户快速浏览。

3. **食谱探索页面(recipes.html)**
   在此页面中，进一步展示一些代表菜品的食谱，并为用户提供做菜灵感“今天吃什么”。采用图文结合的方式，并通过列表或卡片形式组织信息，增强可读性。

4. **食材实验室页面(lab.html)**
   在此页面中，用户可以选择偏好的菜品进行做菜模拟试验，具体学习做一道菜的步骤和要点。

整体信息结构遵循从“总体概览”到“具体内容”的层级逻辑，避免信息堆叠，使用户可以循序渐进地获取信息。

## 四、主要交互功能介绍
1. **首页（index.html）**
深色模式切换：右上角按钮一键切换日间/夜间主题。
回到顶部按钮：右下角悬浮按钮，滚动显示点击平滑返回顶部。
导航栏动态效果：滚动时透明度变化，悬停有下划线指示。
轮播图交互：支持自动播放、手动切换、指示器控制及触摸滑动。
搜索功能：支持关键词搜索和回车提交，前端模拟搜索反馈。
卡片悬停效果：鼠标悬停时卡片上浮、阴影增强并有边框高亮。

2. **地方菜系页面（regional.html）**
对话系统：点击摊位开启多轮对话，结束后显示详情入口。
菜系详情弹窗：展示菜系图文详情，支持图片切换和菜品浏览。
日夜模式切换（集市）：全屏切换日市/夜市场景，改变天空和灯光效果。

3. **食谱探索页面(recipes.html)**
随机推荐卡片：点击翻转卡片随机展示菜品信息并支持换一换功能。
智能筛选系统：按时间/难度/食材/口味多维度筛选菜品。
食谱详情页：折叠式步骤教学，用户可选择感兴趣的食谱一键加入收藏或者一键复制食谱。


4. **食材实验室页面(lab.html)**
拖拽验证：将食材拖入锅中按正确顺序验证的烹饪学习游戏。
步骤引导：分步骤提示烹饪顺序，实时验证正确性的教学系统。
智能反馈：根据操作正确性显示对应弹窗、得分和鼓励信息的反馈机制。
进度跟踪：实时显示挑战进度、得分统计和已使用食材记录的进度系统。

所有交互功能均使用原生 JavaScript 编写，未依赖第三方框架，以确保对基础前端逻辑的理解与掌握。

## 五、Prompt 使用情况简述

在项目开发过程中，明确使用了大型语言模型辅助完成网页代码的生成。首先通过初始 Prompt 描述网站主题、页面结构和基础功能需求，生成 HTML、CSS 和 JavaScript 的初始版本代码。

在此基础上，根据实际效果与课程要求，对生成的代码进行了多轮人工修改与补充，并通过追加 Prompt 的方式优化页面结构、样式细节和交互逻辑。最终项目并非完全由模型自动生成，而是结合了模型输出与个人理解后的调整结果。

相关 Prompt 内容与迭代过程已整理并单独存放于 `docs` 目录中。

## 六、网站运行与测试说明

本项目为纯前端网站，无需后端环境支持。运行方式如下：

1. 解压项目文件夹；
2. 使用浏览器直接打开 `index.html` 文件；
3. 通过导航栏或页面内按钮浏览各页面与交互功能。

在常见桌面浏览器（如 Chrome、Edge）以及移动端浏览环境下进行了基本测试，页面布局与主要交互功能均可正常运行，未发现严重显示或功能错误。
