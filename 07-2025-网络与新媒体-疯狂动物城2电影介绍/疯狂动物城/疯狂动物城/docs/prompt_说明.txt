疯狂动物城2 网站项目 Prompt 说明

一、核心主题与基础样式

1. 主题色彩体系：以蓝色为主色调，通过 :root 变量定义全局色彩，形成统一视觉风格，且支持灵活切换。

◦ 主色 --theme-primary: #1e88e5（浅蓝）：用于导航栏、按钮、特色区块背景等核心交互/视觉元素。

◦ 辅助色 --theme-secondary: #1565c0（深蓝）：用于 hover 状态、强调色，形成色彩层次感，提升视觉区分度。

◦ 中性色：背景色 #f8f8f8、文本色 #333、卡片背景 #fff，保证内容可读性与页面清爽感。

2. 基础重置与全局样式：

◦ 用 * { margin:0; padding:0; box-sizing:border-box; } 统一盒模型，消除浏览器默认边距，避免布局偏差。

◦ 统一 body 字体（微软雅黑等中文字体）、字号（16px）、行高（1.6），确保文本显示一致性，提升阅读体验。

二、核心布局设计

整体采用“固定容器+弹性布局（flex）”为核心，适配多模块排版，布局逻辑清晰：

1. 容器结构：通过 .container 固定 1200px 宽度并居中，搭配阴影效果，形成网页核心内容区，与页面背景区分开。

2. 模块布局：

◦ 横向排版：导航栏（.nav-list）、景点列表（.scenic-list）、特色列表（.feature-list）等采用 flex 布局，支持自适应换行（flex-wrap: wrap），合理分配空间。

◦ 图文组合：关于我们（.about-content）、景点详情（.gallery-item）、新闻（.news-item）等采用“图文左右/反向布局”，通过 flex 分配图文占比，视觉平衡且信息传递高效。

◦ 网格排版：照片画廊（.photo-gallery）按 20% 宽度均分，景点网格（.attractions-grid）按 33.33% 宽度均分，实现密集型内容整齐排列。

三、关键模块样式亮点

1. 导航栏：高度固定 60px，菜单项均匀分布，hover 时背景色切换为深蓝，过渡效果（transition）平滑，提升交互质感。

2. 视觉卡片：景点、新闻、内容卡片等统一使用 border-radius（8-10px）、box-shadow 营造精致感，hover 时触发缩放（scale）或上浮（translateY）效果，增强页面活力。

3. 轮播图：定位+flex 实现轮播容器，左右箭头+底部圆点导航，hover 时箭头背景色加深，激活圆点用主色标记，交互逻辑清晰。

4. 特色区块：.highlight-section 采用渐变色背景，搭配半透明卡片，突出核心数据/信息，视觉冲击力强。

5. 表单样式：输入框、按钮统一圆角设计，输入框聚焦时边框变色，按钮 hover 时背景色加深，提升表单操作体验。

四、交互功能样式设计

内置多种实用交互功能样式，无需额外大量代码即可适配功能开发：

1. 搜索功能：渐变背景搜索区，输入框+按钮采用大圆角设计，聚焦时边框变色，搜索结果区默认隐藏，适配搜索逻辑触发显示。

2. 主题切换：居中排列切换按钮，active 状态用主色标记，hover 时有背景色反馈，支持快速切换网页主题。

3. 展开/折叠：切换按钮居中显示，详情区展开时触发渐入动画（fadeIn），过渡自然，优化长内容阅读体验。

4. 表单反馈：提交成功后显示渐变绿色提示区，搭配滑入动画（slideIn），直观反馈操作结果。

5. 图表控制：图表切换按钮样式与主题切换统一，保持视觉一致性，适配数据可视化模块的交互需求。

五、响应式适配

针对移动设备（max-width: 768px）做关键适配，避免布局错乱：

• 搜索框、主题切换、图表控制改为垂直/换行排版，节省横向空间。

• 图表容器减少内边距，图表自适应宽度，保证在小屏上正常显示。

六、样式规范性与可维护性

1. 样式覆盖逻辑：对不同模块下的相同标签（如 h3）做针对性覆盖，避免样式冲突，例如 .content-card-info h3 清除默认背景与宽度，适配卡片内标题需求。

2. 注释清晰：按“基础样式、模块样式、交互样式、响应式”分类注释，便于后续修改与扩展。

3.变量复用：核心色彩通过 :root 变量定义，后续修改主题色只需调整变量值，无需逐行修改，维护效率高。

七、个人总结
在此次实践中，我们先梳理网页核心主题、布局、模块样式等关键需求，按逻辑拆解为色彩体系、布局设计、交互功能等板块，形成条理清晰的Prompt。通过明确变量定义、布局规则、交互效果等细节，AI 能快速生成符合我们需求的代码。
通过使用这种模式我们的效率大大提高，不用一个代码一个代码的敲来编写基础样式。然后是容易修改，通过改变变量，就很方便修改主题、调整样式。最后是对于我们这种非计算机专业的学生较为方便，因为其专业门槛相对于没那么高，通过提示词即能引导 AI 生成规范代码。同时也遇到了问题：对细节描述要求高，AI 生成的代码需额外修改，同时部分复杂交互逻辑仅靠文字描述难以精准传达，需反复优化表述。比如在初期未明确卡片交互的具体参数，导致生成的效果过于夸张。然后我们的解决方式是补充细节，明确“卡片hover时 scale 为 1.05、translate Y为-3px”，限定效果幅度等。
总体而言，这种prompt模式驱动是一种相对高效的手段，对于我们这种网页设计初学者很友好。
